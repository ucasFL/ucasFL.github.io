数据结构之绪论

发表于 2017/02/28 | 分类于 Data-Structures | 阅读次数

数据

所有能输入计算机并被计算机符号处理的符号，如数值、图形、图像、文字等。

  • 数据元素：数据的基本单位；
  • 数据项：数据不可分割的最小单位；一个数据元素可由若干个数据项组成；

数据结构

存在一定关系的数据元素的集合。

数据元素之间关系的四种基本结构

  • 集合:元素间除同属相同集合外，无其他关系；
  • 线性结构：一对一关系；
  • 树形结构：存在一对多的关系；
  • 图状结构或网状结构：存在多对多的关系；

数据结构的形式定义

二元组： Data Structrue = (D,S);

其中， D 是数据元素的有限集， S 是 D 上关系的有限集。

数据结构的存储方式

  • 存储内容包括数据元素的存储和元素间关系表示存储；
  • 关系的两种表示方法：顺序表示和非顺序表示;

两种不同存储结构：顺序存储结构、链式存储结构；

数据类型

一个值的集合和定义在该值集上的一组操作的总称；

抽象数据类型（Abstract Data Type）(简称 ADT)

一个数学模型和定义在该模型上的一组操作；

ADT 的特征

  • 数据抽象
  • 数据封装

ADT 的形式化定义

ADT = (D,R,P)
其中， D 是数据对象； R 是 D 上的关系集； P 是对 D 的基本操作集；

算法

  • 有穷性
  • 确定性
  • 可行性
  • 输入
  • 输出

算法时间复杂度 - T(n)

与算法中基本操作执行的次数有关；

$T(n)=O(f(n))$ : 存在$M\ge0$，使得当$n\ge n_0$时，$|T(n)|\le M|f(n)|$

空间复杂度 - S(n)

算法编成程序以后，在计算机中所需存储空间大小的度量；

存储空间一般包括：

  • 指令常数变量所占存储空间
  • 输入数据所占存储空间
  • 辅助存储空间
