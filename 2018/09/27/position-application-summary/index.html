两次失败的校招笔试总结

发表于 2018/09/27 | 分类于 Algorithms | 阅读次数

开学的时候，随手投了几分校招。虽然一年后很大的可能还是会继续读研，不过参加一下校招不见得是什么坏事，可以当作是一次简单的考试，同时可以看看自己现在什么水平。

最开始收到的是腾讯的笔试通知，于是大概两周多前参加了腾讯的校招笔试，昨天晚上又参加了百度的校招笔试。两次考试自然都是凉了，不过通过这两次笔试，让我看到了自己身上的很多问题：基础知识掌握的不够好，算法能力实在是太太太太弱了。

笔试都是分为两部分，第一部分是基础题（选择题），第二部分是算法编程题。不过腾讯的基础题和百度的略有不同，算法题考的方向也不太一样。首先说明，我投的都是后端的岗位。腾讯的基础题覆盖的知识点几乎都是数据结构、操作系统、计算机网络以及 C++ 的知识，不过考的确实比较细，需要把所有相关知识掌握牢固才行。比如其中一个题，问的是什么置换算法会产生Belady现象，看到这个题的时候，我蒙圈了，虽然页替换算法操作系统课学过，但我从来没看到什么Belady现象。考试结束后查了一下才知道，原来操作系统课上是有讲到的，只是自己忽视了：Belady现象就是指在 FIFO 替换算法中，增加页的数目，不一定能够减少缺页。

百度的基础题和腾讯略有差别，基础题分为三部分，第一部分是逻辑分析题，第二部分是数学题，第三部分才是关于计算机后端的题。当然，前面的逻辑分析题和数学题都很简单，对于一个理工科专业的学生来说，应该都没有什么问题，只是因为有时间限制，所以容易出现做不完。

下面着重来看一看算法编程题，毕竟这是我的心头之痛。两次笔试的算法题都是有三道，我们逐一来看看。由于具体题目描述已经记不清了，所以下面的题目描述中，仅做形式化描述，并非原题目的完整描述，但不改变题意和要求。

腾讯

1.给定一个正整数 N，求最小的 M，满足 lcm(N+1, N+2, …, N+M) = lcm(1, 2,…, M)

本题一看就是一个数论题，对于一个数学渣来说，数论真的好难啊，然而不幸的是，腾讯的三个算法题中两个是数论题。考试的时候，我自然是没有做出来。下面提供一种暴力解法：M 个数的最小公倍数，可以通过质因数分解来求解，那么便可以将 M 从 1 开始，不停往上加，求 lcm(N+1, N+2, …, N+M) 和 lcm(1, 2, …, M) 直到相等。如果 M 很小，那么能够很快算出来，但如果很大，那么复杂度会很高，必定会超时。

2.给定一个有向图，定义 F(A) 为从顶点 A 出发能够到达（存在路径）的所有顶点的数目，T(A) 为能够到达 A 的所有顶点的数目。给定一个有向图，找出满足 T(A) > F(A) 的顶点数目

本题是一个图论题，原题目是说找”重要城市“的数目，我进行了形式化描述。下面提供一种可行解法：对于任意一个顶点，要计算 F(A)，那么从 A 点开始做一次广度优先遍历即可求出，假设得到的能够到达的顶点集合为 S(A)。对所有顶点都做一次广度优先遍历之后，那么我们便知道了从每一个顶点出发能够到达的顶点，也就可以计算出 T(A) 了，T(A) = A 在所有顶点集合 S(i) 中出现的次数。

分析一下算法时间复杂度，广度优先遍历的时间复杂度是 O(|V|)，|V| 为有向图的顶点数目，对每个顶点都进行一个遍历，因为复杂度为 O(|V|^2)。计算 T(A) 时，S 集合通过索引存储，那么在一个集合中查找需要的时间为 O(1)，需要在 |V| - 1 个集合中查找，需要时间 O(|V|)，每个顶点都需要在 |V| - 1 个集合中进行查找，因此，时间复杂度为 O(|V|^2)。

综上，该算法的时间复杂度为 O(|V|^2)。

百度

下面看一看百度的算法题，和腾讯的相比就没有那么”数学“了，虽然考试的时候我还是没有能够完整做出来一道。再次说明，以下的题目描述均为个人在不改变题意和要求的情况下的形式化描述。

1.给定 n 个数 M1, M2, …, Mn，满足 Mi >= 0，以及两个数 P, Q 满足 P > Q > 0，定义对 n 个数的一次运算 F(M1, M2, …, Mn) 为从这 n 个数中任意选择一个数，假设为 Mi，将 Mi 的值减去 P，如果 Mi <= P，则 Mi = 0，同时将另外 n - 1 个数的值分别减去 Q。问：给定 n 个数以及 P，Q，至少需要多少次 F 运算，能够让所有 n 个数均变为 0?

在该问题中，我们每次可以任意选择一个数，由于 P > Q，因此，被选中的那个数减小值最大，显然，每次都选择 n 个数中最大的那个能够使需要的次数最少，即贪心能够达到最优。解法就是：从 n 个数中选择最大数，将该数的值减小 P，再将另外 n - 1 个数的值分别减少 Q，然后再次选择最大数，重复进行，直到 n 个数都变为 0 为止。

下面，我们对算法的具体实现和复杂度进行一些讨论。第一种简单实现： n 个数顺序存储，每次查找最大数，然后进行一次 F 操作，查找最大数需要的时间为 O(n)，进行一次 F 运算需要的时间也为 O(n)，最坏情况下，需要 Mmax 次 F 运算，Mmax　为初始时 n 个数的最大值。那么总共需要的时间为 Mmax*(n + n) 即 O(nMax)。

对于 F 运算，不太有什么好的办法来优化计算时间，但是对于查找过程，我们可以进一步优化。上面的查找是顺序查找，如果将 n 个数排成有序的，那么我们就可以直接得到最大数，这样查找时间就大大减小了。但是进行了一次 F 运算之后，上一轮的所有非最大数依旧是有序的，而上一轮的最大数可能不再是最大数，则需要进行调整，，将它插入到合适的位置。该算法的详细描述和复杂度分析如下：

  1. 对 n 个数进行一次排序，假设使用快排，需要的时间为 O(nlogn)
  2. 检查最大数是否等于 0，如果是，则结束，否则进行一次 F 运算，需要的时间依然为 O(n)
  3. 进行调整，通过折半查找将上一轮中的最大数插入到合适的位置，查找时间为 O(logn)，然后返回第 2 步。

在该实现中，需要的时间为 nlogn + Mmax logn + n Mmax，第一种实现需要 2n*Mmax，由于 Mmax 的不确定性，如果 Mmax < logn，那么通过排序的方法并不能减少算法需要的时间，因为，需要花费大量的时间进行排序。另外，上面的调整过程只计算了二分查找的时间，没有计算移动元素的时间，最坏情况下，每次需要移动 n-1 个元素，则需要的时间将会远远超过不排序的方法。

实际上，我们可以有更好的改进方法。在数据结构课上，我们学过堆排序，因此我们可以利用堆排序的方法来优化查找过程：

  1. 建一个大顶堆，需要的时间为 O(n)
  2. 取最大数，如果最大数为 0，则结束，否则进行一次 F 运算，需要的时间为 O(n)
  3. 将堆再次调整为大顶堆，需要时间为 O(logn)，返回 2

此时，总共需要的时间为 n + Mmax logn + n Mmax，此时，虽然依旧不一定能够总是比第一种情况好（n + Mmax logn > nMmax时），但是，当 Mmax < n / (n - log n) 时，则能够减少算法需要的时间。

2.给定一个非空 0,1 字符串和一个正整数 K，可以将字符串中的最多 K 个 0 翻转为　１，求翻转后能够得到最长连续 1 的翻转方案的数目

为了解释题目，举个例子，0,1串为 00100，K为 2,即可以将其中两个 0 翻转为 1，则显然能够得到的最长连续 1 的长度为 3，并且翻转方案有:11100, 01110, 00111共三种方案，因此，结果应该输出 3。

该题应该立刻想到动态规划，下面是我想到的一个可行算法，也许还有更好的方案。

假设串为S[n]，定义F(i)为翻转后以第 i 个字符结尾的最长连续 1 子串的长度，则有

  • F(0) = 1
  • i > 0
    • if S[i] == ‘1’，F[i] = F[i-1] + 1;
    • else if S[i] == ‘0’ && numOf0[0~i] <= k,F[i] = i + 1;
    • else F[i] = i - Loc0of[numOf0[0~i] - k - 1];

在计算的过程中，需要记录 S[0] 到 S[i] 之间0的个数以及每一个 0 的 index，计算 F[i] 需要的时间为 O(n)，另外，我们需要在计算的过程中记录最大长度以及最长长度的方法数目。
